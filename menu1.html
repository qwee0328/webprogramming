빅데이터란?

빅데이터 정의

빅데이터는 우리가 매일 사용하는 컴퓨터와 모바일 기기, 기계 센서등 디지털 환경에서 발생하는 대량의 모든 데이터를 의미한다. 기존의 데이터베이스 관리도구의 수집·저장·분석·처리 능력을 넘어서는 방대한 양의 정형 또는 비정형 데이터를 포함하고 있으며 빅데이터는 단지 양이 많다는 점 뿐만 아니라 특성의 다양성과 복잡성으로 인해 "Big"으로 정의된다. 빅데이터는 빅데이터 플랫폼을 구성하는 하드웨어, 소프트웨어, 애플리케이션 간의 유기적 순환에 의해 가치를 창출하고 결과를 분석하는데 사용된다. 또한 넓은 의미로 대규모의 데이터를 저장·관리·분석할 수 있는 하드웨어 및 소프트웨어 기술, 데이터를 유통·활용하는 모든 프로세스를 포함하여 의미하기도 한다.
빅 데이터는 특히 새로운 데이터 소스에서 나온 더 크고 더 복잡한 데이터 세트이다. 데이터를 수집하는 것뿐만 아니라 수많은 데이터 중에서도 필요한 데이터만을 골라 이를 적절하게 활용할 줄 아는 능력이 중요해졌다.
기업에서 흔히 말하는 "빅데이터를 도입한다"라는 것은 조직의 내/외부에 존재하는 다양한 형태의 데이터를 수집, 처리하고 분석하여 데이터에서 의미있는 가치를 추출해 조직에 전략적 의사결정에 활용하고, 비즈니스에 적용하는 행위라고 할 수 있다.

빅데이터 특징(3V)

<초기 빅데이터의 특징>

데이터 세트가 크다고 꼭 빅데이터는 아니다. 빅데이터가 되려면 최소한 아래 3가지 특징이 만족해야 한다.

  • 크기(Volume)
  • 다양성(Variety)
  • 속도(Velocity)
크기(Volume) 데이터 양이 빅데이터를 "Big"으로 정의하는 유일한 요소는 아니지만 중요한 특징이다. 빅 데이터의 사이즈는 단일 데이터 집합의 크기가 수십 테라바이트에서 수 페타바이트에 이른다.
다양성(Variety) 데이터의 다양한 종류를 의미하며 정형화의 종류에 따라 정형, 반정형, 비정형으로 분류할 수 있다. 정형 데이터로만 구성된 데이터 집합은 아무리 양이 커도 반드시 빅데이터가 되지는 않으며, 비정형 및 반정형 데이터 유형은 의미를 도출하고 메타 데이터를 지원하기 위해 추가로 전처리가 필요하다.
속도(Velocity) 빅데이터는 크기가 끊임없이 빠르게 변화하며 대용량의 데이터를 빠르게 처리, 분석, 구성할수 있는 소비속도를 갖는다. 융복합 환경에서 디지털 데이터는 매우 빠른 속도로 생산되며 이를 실시간으로 저장, 유통, 수집, 분석처리가 가능한 성능을 의미한다.

<빅데이터의 새로운 특징>

빅데이터를 통한 가치창출이 중요해지면서 정확성과 가치를 추가한 5V로 나타내기도 한다.

정확성(veracity) 데이터는 정확성과 적시성을 지닌 경우에만 가치가 있으나 정보의 양이 많아진 만큼 데이터의 신뢰성이 떨어지기 쉽다. 따라서 수집한 데이터가 정확한 것인지, 분석할 만한 가치가 있는지 등에 대한 필요성이 중요해졌다. 정형 데이터로만 구성된 기존 데이터베이스는 데이터 정확성과 관련한 구문오류나 오타가 일반적이였으나 비정형 및 반정형 데이터는 완전히 새로운 정확성의 문제가 있다. 사람의 편견, 소셜 노이즈, 데이터 출처의 문제는 모두 데이터 품질에 영향을 미치게 된다.
가치(value) 기업에 있어 빅데이터 분석은 경쟁우위와 회복탄력성을 확보하는 중요한 자원으로 활용될 수 있다. 최근의 빅데이터 기술은 수익과 운영 회복탄력성 모두에 측정가능한 도입효과를 제공할 수 있는 데이터의 수집과 검색 기회를 제공한다.

빅데이터 유형

  • 정형 데이터
  • 비정형 데이터
  • 반정형 데이터
정형 데이터 고정된 필드에 저장되는 일정한 형식의 데이터로 조직화와 검색이 가장 편리하다. 행과 열의 레이아웃이 사전 정의된 엑셀 스프레드시트가 가장 대표적인 예이다. 정형 데이터는 규모가 매우 크더라도 관리가 간단하기 때문에 반드시 빅데이터라고 할수는 없다.
비정형 데이터 고정된 필드에 저정되지 않는 데이터를 의미한다. 소셜미디어 게시글, 오디오 파일, 이미지, 주관식 고객의견 등이 대표적인 예이다. 이러한 유형의 데이터는 표준적인 관계형 데이터베이스로 포착하기 쉽지 않아 데이터 레이크, 데이터 웨어하우스, NoSQL DB에 저장된다.
반정형 데이터 데이터가 저장되는 일정한 구조는 없으나 구조를 파악할 수 있는 데이터를 의미한다. 발신자, 수신자, 제목, 날짜 등 구조화 된 속성이 포함되는 전자 메일이 대표적인 예이다. 지리 태그, 타임스탬프, 시맨틱 태그를 사용하는 장치도 비정형 콘텐츠와 함께 정형 데이터를 제공할 수 있습니다.

빅데이터 소스

  • 소셜 데이터
  • 머신 데이터
  • 거래 데이터
소셜 데이터 소셜 미디어 댓글, 게시글, 이미지, 동영상 등에 의해 생성되는 데이터를 의미한다. 소셜 미디어와 사용 트랜드는 예측할 수 없는 방향으로 빨리 변화하는 경향이 있지만 디지털 데이터 생성 장치의 수는 꾸준히 증가할 전망이다.
머신 데이터 사물 인터넷 기기와 기계에 장착된 센서를 등에 의해 생성되는 데이터를 의미한다. 기업은 IoT 센서를 통해 장비, 차량등의 머신 데이터를 수집, 처리할수 있으며 날씨와 교통, 보안 분야에 이르기까지 전 세계적으로 데이터를 생성하는 사물의 수가 빠르게 증가하고 있다.
거래 데이터 구매 및 금융 거래 등에 의해 생성되는 데이터를 의미한다. 세계에서 가장 빠르게 이동하고 증가하는 데이터이며 엄청난 양의 데이터 생성량을 가진다. 거래 데이터는 반정형 데이터 형식으로 관리와 처리가 복잡하다.

빅데이터의 작동 방식

  • 빅데이터 수집
  • 빅데이터 저장
  • 빅데이터 처리 및 분석
빅데이터 수집 대부분의 빅데이터는 방대한 비정형 및 반정형 데이터 세트로 구성되며 일관되지 않은 개별 소스로부터 제공된다. 기존의 디스크 기반의 데이터베이스 및 데이터 통합 방식은 데이터 취급 작업이 기본적으로 빅데이터와 다르다. 빅데이터를 관리하기 위해서는 빅데이터 획득을 위한 인메모리 데이터베이스 솔루션과 소프트웨어 솔루션 방식이 필요하다.
빅데이터 저장 이름에서 알 수 있듯이 빅데이터는 볼륨이 매우 크다. 많은 기업들은 기존 데이터를 위해 보유하고 있는 솔루션을 이용해 빅데이터의 처리 요구사항을 해결하기를 희망하지만 빅데이터는 크기와 메모리 제한에 따른 제약을 받지 않을 때가 가장 효과적이다. 따라서 빅데이터를 위한 저장 공간과 솔루션이 별도로 필요하다.
빅데이터 처리 및 분석 빅데이터를 실시간으로 일괄 처리하기 전에 일종의 데이터 정제 과정이 먼저 필요하다. 데이터 셋에서 원하는 부분만 혹은 전체를 추출하고 분석을 위해 데이터 세트를 재배치하는 등의 데이터 세트의 폼을 바꾸는 과정을 거친다.
빅데이터 분석에 AI와 머신러닝 기술을 적용하지 않으면 모든 기능을 완전히 실현할 수 없다. 빅데이터의 특징중 하나인 속도를 만족하기 위해서는 분석 프로세스가 정기적으로 자체 최적화되고 경험에서 학습을 할 수 있어야하며 이는 AI기능과 최신 데이터베이스 기술로만 달성할 수 있다.

빅데이터 중요성

빅데이터의 진정한 가치는 데이터를 분석하고 이해할 수 있는 정도에 따라 측정됩니다. 다양한 종류의 대규모 데이터에 대한 생성, 수집, 분석, 표현하는 빅데이터 기술의 발전은 다변화된 현대 사회를 더욱 정확하게 예측하여 효율적으로 작동하게 한다. 뿐만 아니라 인공지능, 머신러닝 기술을 활용하여 빅데이터 분석과 시각화 기술로 현대사회 구성원 마다 맞춤형 정보를 제공, 관리, 분석하여 실행 가능한 실시간 인사이트를 제공할 수 있다. 또한 기업들은 새로운 기회를 실현하고 비즈니스 모델을 구축하는데 도움을 받는다.
